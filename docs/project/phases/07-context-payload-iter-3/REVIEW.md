# Review: Phase 7 - Context Payload Iter-3 + Behavioural Diagnostics

## Review Summary
- **Overall assessment: Concern - approve only after plan edits.**
- **What is solid:** The substrate intent is correctly understood: Vision shrink, Status evac flag, own-speech split, inbound speech restoration, verbose loot outcomes, chest coord ids, and no-op metric redefinition all align with mental-model pillars 4, 5, and 6.
- **What is risky or unclear:** The proposed slim query drops fields that the diagnostics and closing gates still need; retry recovery is not persisted today; the chest-id rename surface omits runtime consumers; and the preferred in-Convex closing path is not the lowest-risk default.

The plan should not go to implementation as-is. The fixes are bounded, but they need to be written into `README.md` before engineers split work.

## Spec / Guide Deviations
- The slim-query proposal currently violates the diagnostics intent because it removes data needed for required metric families while not replacing it with derived lean signals.
- Retry recovery is listed as a required critical-fails metric, but the persisted trace shape does not contain retry-attempt state.
- The "chest ids engine-wide" requirement is not fully reflected in the runtime file surface; `runStats` is especially load-bearing because the equip gate depends on it.
- The loot-outcome "verbose and honest" contract is not complete until same-turn chest collisions and empty chest results produce trace entries.
- The preferred closing path should follow the phase-6 proven local-compute pattern unless a Convex action orchestration is explicitly spiked and shown not to share the parent budget.

## Final Verdict
**Approved with specific edits required.** The plan is directionally aligned with the North Star, but the implementation crew needs an amended data-contract section before starting B/C/D. The most important edits are: add lean derived context signals to `byMatchSlim`, decide/persist retry telemetry or remove that metric, expand the chest-id runtime surface, make same-turn chest loot failures explicit, and default the closing report to local compute plus small-payload persist.
